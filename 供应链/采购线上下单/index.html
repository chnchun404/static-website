1688线上下单

供应链批量采购RPA自动化方案 | 模块化设计 · 精准匹配 · 高效下单

一、业务背景

供应链部门需对SKU进行批量采购下单，下单时需将采购备注中的配送信息与1688下单界面的收货地址匹配并选中，地址无误后完成下单。

批量采购场景下人工处理耗时久、效率低，因此需通过模块化流程自动化，减少人工操作、提升下单效率、确保数据准确性、降低出错率。

二、功能模块

1. 登录马帮ERP系统
  • 打开浏览器访问马帮ERP系统：https://www.mabangerp.com/index.htm
  • 输入《配置文件.xlsx》中的账号密码完成登录
  • 登录失败时读取重试次数限制，仍无法登录则报错并同步至钉钉
2. 导入批量采购模板
  • 进入系统路径：供应链 → 采购计划 → 添加/导入/导出
  • 选择「采购商品导入」，上传采购文件（如《xxxx.xx.xx采购计划导入(一仓）.xlsx》）
  • 导入失败则流程停止并通知钉钉
3. 生成采购单
  • 选择「未采购」状态的商品，全选后点击「生成采购单」
  • 提交订单，并跳转至采购列表查看生成结果
4. 1688下单处理
情况1：采购备注中无配送信息或无法匹配
  • 跳过该记录，将「采购记录为空」或「配送信息匹配失败」写入回执文件
情况2：采购备注中有配送信息且匹配成功
  • 复制配送信息至订单备注
  • 自动匹配1688收货地址，匹配失败则取消交易，继续处理下一条
  • 设置订单参数：红包抵扣为「否」，订单类型为「批发价订单」，交易方式默认「账期交易」，非账期交易则使用「诚E赊」
  • 确认下单，并将下单状态写入回执文件
5. 回执文件生成
  • 下单完成后自动生成回执文件
  • 记录每条SKU的下单状态及失败原因（如「采购记录为空」、「配送信息匹配失败」等），用于后续人工核对

三、异常处理机制

1. 登录失败处理

  • 读取《配置文件.xlsx》中的重试次数限制
  • 重试次数内仍失败则流程停止，错误信息同步至钉钉

2. 导入采购文件失败

  • 采购文件导入失败则流程立即停止
  • 失败信息同步至钉钉通知相关负责人

3. 采购备注信息异常

  • 备注为空或无法匹配配送信息，直接跳过该条记录
  • 回执文件中标记对应失败原因

4. 1688下单失败

  • 收货地址与配送信息不匹配，自动取消交易
  • 不执行下单操作，继续处理下一条记录
  • 将「地址匹配失败」写入回执文件

四、数据传输规范

1. 运行输入

采购计划文件路径（如：《xxxx.xx.xx采购计划导入(一仓）.xlsx》）

2. 运行输出

  • 回执文件：记录每条SKU的下单状态及失败原因
  • 钉钉消息通知：将流程运行记录发送至指定联系人/群聊

3. 数据格式规范

  • 采购计划文件：Excel格式（.xlsx）
  • 回执文件：Excel格式（.xlsx），包含以下列：
    • SKU编号
    • 是否下单成功（「是」或「否」）
    • 失败原因（如「采购记录为空」、「配送信息匹配失败」等）

五、系统依赖

系统名称 类型 是否有账号密码 验证码类型 最近是否有升级计划 备注说明
马帮ERP 浏览器网页 是 无 否 用于导入采购模板、生成采购单、1688下单操作
WPS（Excel） 客户端 否 无 否 用于读取采购计划文件及生成回执文件
钉钉 客户端 否 无 否 用于流程异常通知及运行记录发送
