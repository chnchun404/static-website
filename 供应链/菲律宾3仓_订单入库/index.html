菲律宾3仓-订单入库自动化处理流程

菲律宾对内表格解析 · 验货入库 · 调拨记录生成 · 回执单生成 | 替代人工操作，提升菲律宾3仓订单入库流程效率与准确性

一、业务背景

1.1 实施目的

供应链部门需处理菲律宾3仓（鹏远）采购单入库流程，解析Excel表格（鹏远sheet）中的采购单号、运输方式、入库数量、运单号等信息，完成马帮ERP验货入库、分仓调拨记录生成、流程回执单生成等操作。

人工处理上述流程繁琐且易出错，通过全流程自动化设计，解决人工操作痛点，提升订单入库处理效率与数据准确性。

1.2 基础信息

流程名称 所属部门 操作频率 核心价值
海仓_马帮ERP_菲律宾3仓订单入库 供应链 每日执行（基于当日Excel表格数据） 自动化替代人工，减少重复操作，降低出错率

三、异常处理机制

展开/收起所有异常处理规则
步骤1：马帮ERP系统登录
  • 登录后店铺授权提醒弹窗自动关闭（勾选7天内不再重复提醒）
  • 页面加载超时（800秒）则重建浏览器窗口/重新登录
  • 调拨操作中登录超时自动触发重新登录逻辑
步骤2：菲律宾对内表格数据读取与解析

异常处理规则：

  • 指定文件夹无Excel文件则抛出异常，终止流程
  • 表格中无当日数据则抛出异常，终止流程
  • 识别重复采购单号并标记，重复单号复用已获取的SKU编号
  • 表格无“鹏远”sheet则无法读取核心数据，终止流程
  • 自动清理WPS进程（taskkill /F /IM "wps.exe" /T），避免文件占用
步骤3：马帮ERP验货入库操作

异常处理规则：

  • 采购单号搜索无结果则标记失败，记录失败原因（搜索后暂无内容）
  • 无库存SKU编号则标记失败，需人工确认（该记录无法进行入库，请确认）
  • 点击确认入库后1分钟未出现“入库成功”提示则标记失败（需人工维护）
  • 重复采购单号复用已获取的SKU编号，标记为成功
步骤4：马帮ERP调拨记录生成与发货

异常处理规则：

  • 未完成入库的采购单跳过调拨处理，标记失败（没有完成入库，不进行后续）
  • 搜索调拨记录无结果则标记失败（没有查询到任何信息）
  • 发货备注填写后30秒未出现“操作成功”弹窗则标记失败（分仓调拨备注失败）
  • 发货操作确认弹窗超时（30秒）则标记失败
  • 页面加载超时（800秒）触发重新登录/重建浏览器窗口
  • 调拨发货操作失败则标记异常，记录具体异常原因

四、数据传输规范

展开/收起所有数据传输规则
规则1：菲律宾对内表格数据解析规范
  1. 日期筛选：仅处理表格中“时间”字段为当日的数据（截取日期部分，去除时间）
  2. 仓库匹配：菲律宾3仓（鹏远）
  3. 运输方式适配：“陆运”自动转换为“陆地运输”（适配马帮ERP字段）
  4. 重复处理：相同采购单号+运输方式的记录合并，识别重复采购单号
规则2：流程回执单生成规范
  1. 字段：采购单号、运输方式、入库数量、运单号、SKU编号、仓库编码、入库状态、失败原因
  2. 状态值：成功/失败（失败需填充具体原因，如“搜索后暂无内容”“入库超时”“分仓调拨备注失败”）
  3. 命名规则：{时间戳（含毫秒）}_菲律宾3仓入库回执单.xlsx
  4. 存储路径：指定的输出文件夹，与输入文件夹分离
规则3：文件命名与存储规范
  1. 输入文件：共享文件夹路径（读取指定文件夹下的Excel文件，优先读取含“鹏远”的sheet）
  2. 输出回执单：指定输出文件夹，按时间戳命名避免重复
  3. 临时文件：自动清理WPS进程（taskkill /F /IM "wps.exe" /T），避免文件占用
  4. 浏览器清理：操作前关闭所有Chrome窗口，避免进程冲突
规则4：调拨记录编辑规范
  1. 仓库匹配：菲律宾3仓（鹏远）
  2. 备注格式：在发货备注中填写采购单号对应的运单号
  3. 操作校验：填写备注后校验是否出现“操作成功”弹窗，确保操作生效
  4. 多记录处理：按采购单号精准匹配调拨记录，避免错填备注

五、系统依赖

5.1 基础环境

  • 操作系统：Windows（依赖win32系列库、进程管理、系统命令）
  • 编程语言：Python 3.x

5.2 核心Python库

  • 自动化类：xbot（网页/客户端自动化）、xbot_visual（图像识别/加载状态检测）
  • 数据处理类：pandas（Excel表格解析/回执单生成）
  • 系统操作类：os（文件/目录操作）、subprocess（系统命令执行、进程管理）、time（时间处理）、traceback（异常追踪）
  • 日志类：xbot.app.logging（日志记录）
  • 扩展类：xbot_extensions（马帮登录扩展）
