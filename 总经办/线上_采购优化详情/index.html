线上_采购优化详情自动化处理流程

采购成本优化自动化方案 | 马帮ERP数据筛选 · 采购单批量提取 · 历史数据比对 · 单价差异分析

一、业务背景

1.1 实施目的

企业内需对1688线上采购单进行成本优化分析，核心是提取马帮ERP中「采购中-已申请付款-待付款」的当前采购单，与历史「已完成」采购单按SKU精准匹配，计算**本次与上次采购的实际单价差异、降价金额、差异总金额**，生成标准化单价差异化比对表。

人工完成马帮ERP数据筛选、采购单信息提取、历史数据比对、公式计算等工作，存在效率低、易出错、单价计算不精准、成本优化空间无法量化等痛点，通过自动化流程实现采购数据端到端分析，精准识别成本优化点，提升采购决策效率。

1.2 基础信息

流程名称 所属部门 操作频率 单次耗时
1688线上_采购优化详情 总经办 按需执行（手动触发） 自动化：5~10分钟（人工：40+分钟）

2.1 涉及应用系统

Python运行环境 后台运行 无 数据处理、自动化逻辑执行、异常监控

2.2 核心模块概览

马帮ERP采购数据多条件筛选 马帮ERP 网页数据（内存提取）

2.3 核心自动化能力

  • 马帮ERP自动登出重登，校验当前登录用户，避免账号错登导致数据错误
  • 采购管理页面多条件自动筛选（付款状态：已申请付款/平台状态：待付款），支持时间范围精准过滤
  • 采购单批量提取，按100条/页分页处理，自动计算总页数并批量获取SKU/名称/采购价/运费等核心信息
  • 同SKU历史采购数据精准匹配，自动筛选「日期最新、早于本次采购」的历史记录，避免重复比对
  • 实际单价自动计算（采购价+运费/采购数量），支持整数/浮点型数据自动转换，保留两位小数
  • Excel模板自动填充数据，批量生成计算公式（实际单价/降价金额/差异总金额），并统一数值格式
  • 页面加载状态实时监控，图像识别「加载中」图标，超时自动重试并抛出异常，保证流程稳定性
  • 输出文件自动命名（含时间戳）、指定路径存储，文件生成后校验存在性，确保输出成功

三、异常处理机制

展开/收起所有异常处理规则
步骤1：马帮ERP登录状态校验

异常处理规则：

  • 流程启动前强制终止WPS/Chrome残留进程，避免进程占用导致操作失败
  • 自动执行登出操作后重新登录，校验登录信息与全局变量glv['Login_info']一致
  • 访问采购管理页面后，无法获取「当前用户名」则判定登录失效，抛出明确异常
  • Chrome浏览器以无痕模式启动，加载超时（300s）直接终止流程并提示
步骤2：采购管理页面元素操作

异常处理规则：

  • 所有按钮点击通过CommonUtil工具类实现，最多重试10次，每次间隔1s，确保元素可点击
  • 点击元素前先激活浏览器窗口，避免窗口后台导致点击无效
  • 点击「更多条件/付款状态/平台状态」后，等待加载完成再执行下一步，防止操作串行失效
  • 元素点击超时（10次重试失败）则抛出「元素不存在/点击超时」明确异常
步骤3：采购数据多条件筛选

异常处理规则：

  • 筛选后显示「暂无筛选的采购记录」则直接抛出异常，提示用户检查采购设置
  • 时间范围输入框自动填充（开始/结束时间），非模拟输入确保数据格式正确
  • 重新筛选（时间范围）后，再次校验数据数量，避免筛选条件不生效
  • 分页设置为100条/页，设置失败则按默认页数处理，不终止流程
步骤4：页面加载状态实时监控

异常处理规则：

  • 通过图像识别监控「加载中」图标，最多循环600次（约10分钟），每1s检测一次
  • 加载过程中每2s校验一次，图标消失则立即退出监控，继续流程
  • 600次循环后仍显示「加载中」则判定加载失败，抛出明确异常
  • 所有数据筛选/分页/搜索操作后，均强制执行加载状态监控，避免数据未加载完成
步骤5：采购单信息批量提取

异常处理规则：

  • 获取「总数据量」后，通过向上取整自动计算总页数，避免分页遗漏
  • 单页数据提取失败则跳过当前页，记录警告日志，不终止整批处理
  • 采购单信息（SKU/价格/数量）提取为非预期类型时，保留原始值并尝试转换
  • 批量提取过程中，浏览器意外关闭则捕获异常，记录堆栈信息并终止流程
步骤6：历史与本次采购数据匹配

异常处理规则：

  • 匹配时排除「本次采购记录」（日期+SKU一致），避免自比对
  • 同SKU存在多条历史记录时，自动筛选「日期最新」的记录，确保比对有效性
  • 无历史采购记录则跳过比对，对应列留空，不触发计算异常
  • 采购日期字符串转换为datetime对象失败时，抛出异常并提示数据格式错误
步骤7：采购优化表生成与校验

异常处理规则：

  • 加载Excel模板前，校验模板文件路径glv['Template_File']是否存在
  • 数据填充时，自动处理整数/浮点型转换，除法/加法运算异常时保留原始计算逻辑
  • 文件保存到指定路径后，强制校验文件是否存在，不存在则抛出「文件输出失败」异常
  • 输出文件自动创建「处理完成」文件夹，支持文件夹已存在的容错处理
  • Excel公式生成后，统一设置「0.00」数值格式，避免科学计数法/格式混乱

四、数据传输规范

展开/收起所有数据传输规则
规则1：采购中数据筛选与提取规范
  1. 固定筛选条件：「采购中」状态+「付款状态=已申请付款」+「平台状态=待付款」，不可随意修改
  2. 时间范围自动获取为指定周期，格式为「YYYY-MM-DD 00:00:00」至「YYYY-MM-DD 23:59:59」，非模拟输入
  3. 分页设置为100条/页，按「总数据量/500」向上取整计算总页数，确保批量提取无遗漏
  4. 提取核心字段：库存SKU、中文名称、采购时间、运费、采购价、采购数量，按固定顺序存储
  5. 采购单数量转换为整型，采购价/运费转换为浮点型，确保后续计算精度
规则2：历史已完成采购数据筛选规范
  1. 历史数据筛选状态为「已完成」，与本次采购使用相同时间范围过滤
  2. 按「采购单号」批量提取，确保数据与本次采购单维度一致
  3. 提取字段与采购中数据完全一致，保证后续匹配和计算的兼容性
  4. 历史数据存储为键值对结构，以「采购单号」为键，方便快速匹配
规则3：SKU采购数据匹配规范
  1. 匹配主键为「库存SKU」，确保同商品精准比对，忽略名称细微差异
  2. 时间筛选规则：历史采购时间 < 本次采购时间，排除同期或后期采购记录
  3. 多条历史记录筛选规则：按采购时间降序，取第一条（日期最新），确保比对时效性
  4. 匹配排除规则：SKU一致且采购时间完全相同的记录，直接跳过，避免自比对
  5. 无匹配结果时，历史数据字段留空，不参与后续计算，保证流程连续性
规则4：单价差异化比对表数据填充规范
  1. Excel模板固定为glv['Template_File']指定文件，工作表为第一个（索引0），不可随意替换
  2. 数据从第5行开始填充，保留前4行表头和说明信息，保证模板格式统一
  3. 本次采购数据列：A(库存SKU)、B(中文名称)、C(本次采购时间)、D(本次采购数量)、E(本次采购价)、F(本次运费)
  4. 历史采购数据列：H(上次采购时间)、I(上次采购数量)、J(上次采购价)、K(上次运费)
  5. 数据填充时，空值直接留空，不填充默认值，保证数据真实性
规则5：Excel公式自动生成规范
  1. 本次实际单价（G列）：公式=E{行号}+(F{行号}/D{行号})，采购价+单位运费，统一保留2位小数
  2. 上次实际单价（L列）：公式=J{行号}+(K{行号}/I{行号})，计算逻辑与本次保持一致
  3. 降价金额（M列）：公式=G{行号}-L{行号}，本次实际单价-上次实际单价，负值表示涨价
  4. 差异总金额（N列）：公式=D{行号}*M{行号}，本次采购数量*单台降价金额，量化总优化空间
  5. 所有公式列统一设置「0.00」数值格式，避免计算结果格式混乱，保证报表美观
规则6：输出文件命名与存储规范
  1. 输出文件存储路径：glv['RETURN_FOLDER']/处理完成，自动创建文件夹，支持已存在容错
  2. 文件命名规则：1688线上_采购优化详情_${时间戳}.xlsx，时间戳格式为系统默认，确保唯一性
  3. 文件格式固定为XLSX，兼容WPS/Excel 2016+所有版本，避免格式兼容问题
  4. 文件保存后，强制校验文件存在性，不存在则抛出异常，确保输出成功
  5. 输出文件路径写入全局变量glv['output_file_save_path']，方便后续流程调用

五、系统依赖

5.1 运行环境依赖

  • 操作系统：Windows 10/11 64位（需支持taskkill、os.system命令）
  • 浏览器：Google Chrome 90.0+（支持无痕模式、窗口激活、元素定位）
  • 办公软件：WPS Office 2021+/Microsoft Excel 2016+（支持xlsx格式、公式计算）
  • 运行时：Python 3.8+（需安装指定第三方库，见下方）
  • 自动化框架：xbot/xbot_visual（影刀RPA框架，提供网页/图像/Excel自动化能力）

5.2 Python第三方库依赖

  • 网页自动化：xbot、xbot_visual（影刀核心库）
  • Excel操作：openpyxl>=3.0.10（模板加载、数据填充、公式生成）
  • 数据处理：pandas（可选）、math（内置，向上取整计算）
  • 时间处理：datetime>=4.3（内置，日期转换、比对）、time（内置，延时、等待）
  • 文件操作：os（内置）、shutil（内置，文件路径处理）
  • 日志与异常：xbot.app.logging、traceback（内置，异常堆栈记录）
  • 数据解析：re（内置，正则匹配）

5.3 权限依赖

  • 马帮ERP：采购管理模块、供应链模块的查看/筛选/分页权限，可访问采购中/已完成数据
  • 本地文件：模板文件（Template_File）读权限，输出文件夹（RETURN_FOLDER）读/写/新建权限
  • 系统权限：允许执行taskkill命令（终止残留进程）、允许Chrome浏览器无痕模式运行
  • 网络权限：允许访问马帮ERP官网（https://www.mabangerp.com），网络连接稳定无代理
  • 进程权限：允许终止WPS/Chrome进程，无系统进程保护限制

5.4 配置文件依赖

  • 全局变量：glv（package.variables），核心参数：SETTING_TABLE（配置表）、RETURN_FOLDER（输出根路径）、Template_File（Excel模板路径）、Login_info（马帮登录信息）、output_file_save_path（输出文件路径）
  • Excel模板：Template_File指定的xlsx文件，包含固定表头、列名、数据起始行，不可随意修改结构
  • 工具模块：tools.py（自定义工具类，提供时间获取、批量处理等通用方法）
  • 扩展模块：mb_close_box.py（马帮弹窗关闭）、login.py（马帮登出/登录）
  • 图像资源：「加载中_图像」（xbot_visual图像识别用，用于监控页面加载状态）
